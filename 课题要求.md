# 面向“保险\+医养”生态的跨域知识图谱构建与问答系统

## **一、课题背景**

我国老龄化加快，“保险\+医养”融合成为新趋势。但该领域涉及保险、医疗、养老等多方面知识，存在术语不统一、信息割裂等问题。
大语言模型（LLM）虽能生成自然语言，但在专业问题上容易“说错话”（幻觉）。
GraphRAG（基于知识图谱的检索增强生成） 可以结合结构化知识与大模型优势：用图谱确保事实准确，用大模型生成易懂回答。

## **二、课题目标**

1. 构建跨域知识图谱：整合保险、医疗、养老三类知识，包含疾病、药品、保险产品、养老机构等实体及其关系。

2. 开发 GraphRAG 问答系统：

    - 能理解用户关于健康、保险、养老的复杂问题；

    - 从图谱中检索相关知识；

    - 用大模型生成准确、可读、可溯源的回答；

    - 支持典型问题如：“70岁高血压能买XX护理险吗？”

3. 提升AI工程能力：掌握知识图谱构建、图检索、大模型调用和提示工程等技能。

## **三、课题任务要求**

#### **（一）知识图谱构建**

1. 收集数据：至少用两类以上来源，如：

    - 医疗库（ICD、DrugBank、医保目录）

    - 保险条款（公开或模拟）

    - 养老服务信息（如民政部养老机构名单）

2. 设计本体：定义核心实体（如疾病、保险产品）和它们的关系（如“承保”“治疗”“年龄限制”）。

3. 抽取与对齐：从文本中提取三元组，并统一不同来源的相同概念（如“高血压” vs “原发性高血压”）。

4. 存储与展示：用任意图数据库存储图谱，支持基本查询和可视化。

#### **（二）GraphRAG 问答系统**

1. 理解问题：识别问题中的关键信息（如年龄、疾病、保险类型）和用户意图（是否可投保？费用？）。

2. 图谱检索：将问题中的实体匹配到图谱节点，查找1\-2跳内的相关子图。

3. 生成答案：把检索到的三元组转成自然语言描述，输入给大模型（如 Qwen、ChatGLM），生成最终回答，并注明来源。

4. 提供交互界面：支持网页或命令行提问，返回带引用的答案。

## **四、功能实现要求**

|模块|功能点|说明|
|---|---|---|
|图谱构建|实体/关系抽取、Neo4j 存储|实体/关系抽取、Neo4j 存储|
|图谱检索|基于实体的子图查询|支持1–2跳关系扩展，返回相关三元组|
|GraphRAG 引擎|上下文构造 \+ LLM 调用|将图谱结果转为 prompt，调用 LLM 生成答案|
|大模型集成|本地部署或 API 调用|可使用开源小模型（如 Qwen\-1\.8B\-Chat、ChatGLM3\-6B）或阿里云百炼/Qwen API|
|用户界面|支持问答与答案溯源|显示答案 \+ 引用的图谱三元组或来源|

## **五、技术实现建议（仅做参考）**

1. 知识图谱栈：

    1. 抽取：spaCy / LLM\-based IE（如使用 LLM 从条款中抽三元组）

    2. 存储：Neo4j（支持 Cypher 查询子图）

2. GraphRAG 核心流程：

    1. 用户输入 → 实体识别（如使用 `HanLP` 或 `BERT-NER`）

    2. 实体链接到图谱节点

    3. 执行 Cypher 查询获取相关子图（例如 `MATCH (p:InsuranceProduct)-[r]->(d:Disease) WHERE d.name = '糖尿病' RETURN p, r, d`）

    4. 将查询结果格式化为自然语言上下文（如 JSON → 文本）

    5. 构造 Prompt

    6. 调用 LLM（本地或 API）生成答案

3. 大模型选项（面向学生友好）：

    1. 本地运行：Qwen\-1\.8B\-Chat（4\-bit 量化后可在消费级 GPU 运行）

    2. 云端 API：阿里云百炼平台（免费额度）、智谱 AI、OpenRouter 等

4. 部署框架：FastAPI \+ React/Vue，或 Streamlit 快速原型

## **六、预期成果**

1. 一个包含 ≥500 实体、≥1000 关系的“保险\+医养”知识图谱（Neo4j 格式）；

2. 一个基于 GraphRAG 架构的问答系统原型，支持自然语言提问与带依据的回答；

3. 技术报告需包含：

    - 本体设计图

    - GraphRAG 流程图

    - Prompt 设计示例

    - 对比实验（可选）：纯 LLM 回答 vs GraphRAG 回答（展示事实准确性提升）

4. （加分项）支持答案溯源高亮（如点击答案可查看原始三元组）。

## **七、拓展方向（加分项）**

- 引入 HyDE（Hypothetical Document Embeddings） 提升检索质量；

- 结合 向量数据库 \+ 图数据库 实现混合检索（文本语义 \+ 结构关系）；

- 使用 LLM 自动评估 问答准确性（如基于预设标准答案）；

- 探索 多轮对话 中的图谱状态跟踪（如用户追问“那如果我有肾病呢？”）。
