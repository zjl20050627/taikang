# GraphRAG 问答系统模块

## 目录结构
- **question-understanding/**：问题理解
  - 实体识别模块
  - 意图分类器
  - 关键信息提取
- **graph-retrieval/**：图谱检索
  - 实体链接到图谱节点
  - 子图查询（1-2跳）
  - 检索结果过滤
- **answer-generation/**：答案生成
  - 上下文构造
  - Prompt 设计
  - 答案格式化
- **llm-integration/**：大模型集成
  - 本地模型配置
  - API 调用接口
  - 模型参数调优
- **interface/**：用户界面
  - 网页界面
  - 命令行工具
  - 答案溯源展示

## 功能设计
1. **问题理解**：识别用户问题中的关键信息（如年龄、疾病、保险类型）和用户意图（是否可投保？费用？）
2. **图谱检索**：将问题中的实体匹配到图谱节点，查找1-2跳内的相关子图
3. **答案生成**：把检索到的三元组转成自然语言描述，输入给大模型，生成最终回答，并注明来源
4. **大模型集成**：支持本地部署或 API 调用，可使用开源小模型或云服务
5. **用户界面**：支持网页或命令行提问，返回带引用的答案

## 技术栈
- 实体识别：HanLP / BERT-NER
- 图谱检索：Cypher 查询
- 大模型：Qwen-1.8B-Chat / ChatGLM3-6B / 阿里云百炼API
- 部署框架：FastAPI + React/Vue 或 Streamlit

## 预期成果
- 一个基于 GraphRAG 架构的问答系统原型
- 支持自然语言提问与带依据的回答
- 典型问题如："70岁高血压能买XX护理险吗？"的准确回答
