# 知识图谱构建模块

## 目录结构
- **data/**：数据收集与存储
  - 医疗库数据（ICD、DrugBank、医保目录）
  - 保险条款数据
  - 养老服务信息
- **extraction/**：实体与关系抽取
  - 文本解析脚本
  - 三元组提取工具
  - 实体对齐模块
- **ontology/**：本体设计
  - 核心实体定义
  - 关系类型定义
  - 术语映射表
- **storage/**：图谱存储
  - Neo4j 连接配置
  - 数据导入脚本
  - 查询接口
- **visualization/**：图谱可视化
  - 可视化配置
  - 展示脚本

## 功能设计
1. **数据收集**：从多个来源获取医疗、保险、养老相关数据
2. **本体设计**：定义核心实体（疾病、药品、保险产品、养老机构等）和它们的关系（承保、治疗、年龄限制等）
3. **实体关系抽取**：从文本中提取三元组，并统一不同来源的相同概念
4. **图谱存储**：使用 Neo4j 存储图谱，支持基本查询
5. **可视化**：提供图谱可视化界面，便于理解和分析

## 技术栈
- 数据处理：Python、Pandas
- 实体识别：spaCy / LLM-based IE
- 图谱存储：Neo4j
- 可视化：Neo4j Bloom / Python 可视化库

## 预期成果
- 一个包含 ≥500 实体、≥1000 关系的"保险+医养"知识图谱
- 完整的图谱构建流程脚本
- 图谱可视化界面
